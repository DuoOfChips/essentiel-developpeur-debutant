# Parcours Soft Skills — Autonomie, Communication, Prise de Décision

## Fondations (à travailler en continu)
- [ ] Lire et comprendre un message d’erreur
- [ ] Formuler une question clairement
- [ ] Reformuler ce qu’on a compris

### Exercices
- Expliquer "ce que l’erreur veut dire" avant d’utiliser ChatGPT
- Méthode pour demander de l’aide :
  - Contexte → Objectif → Problème → Hypothèse

---

## Recherche & Apprentissage Autonome
- [ ] Savoir chercher sur Google / StackOverflow
- [ ] Savoir lire une documentation technique
- [ ] Savoir tester une hypothèse en modifiant le code

### Rituel hebdo
1. Rechercher une solution seul 10 minutes
2. Comparer avec la solution de la doc
3. Reformuler la compréhension

---

## Collaboration & Communication Technique
- [ ] Faire des Pull Requests propres
- [ ] Savoir expliquer un choix technique simplement
- [ ] Savoir résumer un problème en 3 phrases (à peu près).

### Modèle de Pull Request
- [ ] Titre clair
- [ ] Contexte
- [ ] Changements effectués
- [ ] Pourquoi ce choix ?
- [ ] Tests / impacts / limites

---

## Discipline & Qualité
- [ ] Refactoriser régulièrement (sans casser)
- [ ] Écrire des fonctions courtes
- [ ] Préférer la lisibilité à l’optimisation prématurée

---

## Esprit Produit & Entrepreneuriat
- [ ] Prioriser les fonctionnalités par valeur d’usage
- [ ] Reconnaître la dette technique et savoir quand l’accepter
- [ ] Décider quand dire “c’est assez bon pour maintenant” (contexte coût/priorité, criticité, temps).
- [ ] Comprendre que rien n'est une évidence (si c'est évident sans confirmation nécessaire : très mauvais signe)

### Outil pratique
- Tableau à trois colonnes :
  - Valeur utilisateur immédiate (0 à 4)
  - Coût technique (0 à 4)
  - Risque futur (0 à 4)

*0 = faible, 4 = élevé*
